# Front Skill Examples

Good output:

- Starts with `<!DOCTYPE html>`.
- Implements the named sections from the spec.
- Uses empty states or user-created local state unless the spec requires preexisting visible data.
- Handles mobile and desktop widths.
- Has visible focus and meaningful labels.

Bad output:

- Explains what it would build instead of returning HTML.
- Uses CDN assets or framework imports.
- Creates a generic hero page unrelated to the user story.
- Omits acceptance criteria or interactions.
- Lets text overlap cards, buttons, or panels.
