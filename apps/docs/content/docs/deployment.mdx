## Application Runtime

The primary supported application runtime today is local pnpm execution:

```bash
pnpm install
pnpm dev
```

Production-like local execution:

```bash
pnpm build
pnpm --filter @u-build/server start
```

The product app and API are not yet documented as a production deployment artifact. Treat local pnpm execution as supported and Docker/production packaging as planned until the Docker specification is implemented and validated.

## Docker Status

Docker support is planned separately. Until Docker artifacts exist and pass validation, Docker should be described as planned, not supported.

Expected Docker direction:

- One app image for web/API or a documented split if needed.
- File-mode volume for `HORUS_DATA_DIR`.
- Optional Postgres profile.
- No secrets baked into images.
- No machine-specific host paths.

## Future Docker Acceptance Criteria

Docker work should be considered complete only when it proves:

| Requirement | Acceptance Signal |
| --- | --- |
| Cross-platform startup | Same documented command works on macOS, Windows, and Linux Docker hosts |
| Portable persistence | File mode uses named volumes or configurable bind mounts |
| Optional Postgres | Compose profile starts API with Postgres-backed repositories |
| Secret safety | `.env` is mounted/injected, never copied into image layers |
| Build reproducibility | Image builds from a clean clone without local caches |
| Health validation | `/health` responds from the containerized API |
| Docs parity | README and docs explain container commands and reset paths |

## Documentation Site

The docs site is a standalone Next.js app under `apps/docs`.

Local development:

```bash
pnpm --filter @u-build/docs dev
```

Production build:

```bash
pnpm --filter @u-build/docs build
```

Recommended deployment target:

- Vercel project root: `apps/docs`
- Framework: Next.js
- Build command: `pnpm build`
- Output: Next.js default
- Node.js: `20` or newer
- Public URL: `https://docs-three-coral.vercel.app/docs`

<Info>
The docs app is intentionally separate from `apps/web` so product UI changes do not destabilize documentation, and documentation UI changes do not affect the product app.
</Info>

## Documentation Release Checklist

Before publishing the documentation site:

- Run `pnpm --filter @u-build/docs build`.
- Smoke `http://localhost:3002/docs` when running locally.
- Confirm `README.md` points to the current deployed docs URL.
- Confirm private local files, runtime state, generated output, and planning-only artifacts are not included in the deployment.

## Vercel Notes

The documentation app is safe to deploy independently because it does not need provider secrets or runtime Horus state. It is a static/Next documentation surface that reads MDX content from `apps/docs/content`.

If the app is deployed from the `apps/docs` directory, the package should install its own Next/Fumadocs dependencies and run its local `pnpm build` script. If it is deployed from the monorepo root, the project configuration must set the docs app as the build target.
