## Purpose

The preview runtime lets Horus launch and inspect frontend projects while workflows and chat-driven changes are in progress. It is operational evidence, not just a convenience UI.

## Preview Project Seed

The default web preview project is configured through environment variables:

| Variable | Purpose |
| --- | --- |
| `HORUS_WEB_PROJECT_ROOT` | Root path for the previewable web project |
| `HORUS_WEB_PROJECT_NAME` | Display name for the seeded project |
| `HORUS_WEB_PROJECT_SLUG` | Stable slug for the seeded project |
| `HORUS_WEB_DEFAULT_ROUTE` | Route opened by default |
| `HORUS_PACKAGE_MANAGER` | Package manager command |
| `HORUS_WEB_PACKAGE_FILTER` | Workspace package filter |
| `HORUS_WEB_PREVIEW_HOST` | Preview host |
| `HORUS_WEB_PREVIEW_PORT` | Preview port |
| `HORUS_WEB_PREVIEW_URL` | Browser-facing preview URL |

If project discovery fails, configure `HORUS_WEB_PROJECT_ROOT` explicitly.

## Session Lifecycle

<Steps>
  <Step title="Register project">
    The frontend project registry returns a previewable project definition.
  </Step>
  <Step title="Create session">
    A preview session records target project, route, device, and initial state.
  </Step>
  <Step title="Start runtime">
    The preview adapter starts the configured command and records process timeline evidence.
  </Step>
  <Step title="Observe timeline">
    Preview events and status changes are surfaced to the UI.
  </Step>
  <Step title="Validate smoke">
    QA/Curator flows can ask for preview smoke evidence.
  </Step>
  <Step title="Stop or recover">
    Runtime process handles are stopped explicitly or recovered as stale after restart.
  </Step>
</Steps>

## Durable vs Runtime State

| State | Durable? | Notes |
| --- | --- | --- |
| Preview session metadata | Yes | Stored in file mode or Postgres mode |
| Timeline events | Yes | Used for debugging and evidence |
| Device/route selection | Yes | User-facing preview state |
| Spawned process handle | No | Runtime-local and recovered as stale after restart |
| Browser-facing URL | Yes | Stored as metadata, but availability depends on runtime process |

## Generated Project Preview

Generated project previews use their own host/port controls:

| Variable | Purpose |
| --- | --- |
| `HORUS_GENERATED_PROJECT_PREVIEW_HOST` | Host for generated project preview |
| `HORUS_GENERATED_PROJECT_PREVIEW_PORT` | Port for generated project preview |

Generated project preview state should be treated as runtime state and should not be committed.

## Failure Modes

| Failure | Operational Response |
| --- | --- |
| Port already in use | Change preview port or stop the conflicting process |
| Package manager missing | Install the configured package manager or change `HORUS_PACKAGE_MANAGER` |
| Command policy blocks execution | Update the project command catalog intentionally |
| Project root cannot be found | Configure `HORUS_WEB_PROJECT_ROOT` |
| Process died after restart | Recover stale session and start a new runtime process |
| Smoke validation fails | Preserve timeline evidence and route feedback to QA/Curator |
