## Prerequisites

| Requirement | Version |
| --- | --- |
| Node.js | `>=20` |
| pnpm | `>=9` |
| Git | Any current release |
| Postgres | Optional |

Recommended baseline:

```bash
node --version
pnpm --version
git --version
```

<Warning>
Do not commit runtime secrets, local runtime state, generated workspaces, dependency folders, or machine-specific operator notes.
</Warning>

## First Run

<Steps>
  <Step title="Enable Corepack">
    ```bash
    corepack enable
    corepack prepare pnpm@9.15.0 --activate
    ```
  </Step>
  <Step title="Install dependencies">
    ```bash
    pnpm install
    ```
  </Step>
  <Step title="Create local environment file">
    ```bash
    cp .env.example .env
    ```
  </Step>
  <Step title="Select local persistence in `.env`">
    Add or confirm these values in `.env`:

    ```bash
    PERSISTENCE_DRIVER=file
    HORUS_DATA_DIR=.horus/data
    ```
  </Step>
  <Step title="Configure an LLM provider in `.env`">
    Choose a provider/model available to your account:

    ```bash
    LLM_PROVIDER=openai
    LLM_MODEL=<provider-model>
    OPENAI_API_KEY=...
    ```
  </Step>
  <Step title="Start development">
    ```bash
    pnpm dev
    ```
  </Step>
  <Step title="Check the API">
    ```bash
    curl http://localhost:3000/health
    ```
  </Step>
</Steps>

## Windows Notes

PowerShell equivalents:

```powershell
corepack enable
corepack prepare pnpm@9.15.0 --activate
pnpm install
Copy-Item .env.example .env
pnpm dev
```

Environment variables can be placed in `.env`; they do not need to be exported in the shell for normal local development.

## Production-Like Local Start

```bash
pnpm build
pnpm --filter @u-build/server start
```

The server reads `PORT`, `HOST`, `PERSISTENCE_DRIVER`, `HORUS_DATA_DIR`, provider variables, and optional database variables from the environment.

## File Mode vs Postgres Mode

Use file mode when:

- You are developing locally.
- You need inspectable JSON state.
- You want the smallest possible setup.

Use Postgres mode when:

- State must survive container replacement or deployment restarts.
- Multiple processes or environments need a shared durable store.
- You need normal database backup/restore workflows.

File mode is portable when `HORUS_DATA_DIR` is configured intentionally and ignored by Git.
